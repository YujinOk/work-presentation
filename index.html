고객을 이해하는 개발자

Developer Presentation
01

1️⃣ 내가 하는 일

역할 & 책임

  • 사주 웹앱 개발 전반 담당
  • 사주 계산 로직 설계 및 구현
  • 프론트엔드 개발 (UX 중심 화면 구성)
  • AI 기반 해석 구조 설계 (룰 + LLM 하이브리드)
  • 사용자 흐름(입력 → 해석 → 결과) 전체 구조 책임

"기획–개발–구현을 하나의 흐름으로 연결하는 역할"

02

2️⃣ 핵심 강점 ① 멀티 역할 수행 능력

강점 1. 멀티 역할 수행 능력

  • 개발자이지만 기획, 고객, 운영 관점 모두 이해
  • 개발 이전 단계에서 문제를 발견하는 타입

사례

  • 기능 정의 단계에서 → 불필요한 기능 제거
  • → 사용자 혼란 가능성 사전 차단

"개발은 구현이 아니라 선택의 연속이라고 생각합니다."

02

2️⃣ 핵심 강점 ② 빠른 적응력 & 전환 능력

강점 2. 빠른 적응력 & 전환 능력

  • 다양한 업무 경험 (CS, 커뮤니케이션, 개발)
  • 도메인 이해 속도 빠름
  • 상황에 따라 우선순위를 빠르게 재정렬

사례

  • 사주 도메인 비전공 상태에서
  • → 단기간에 핵심 구조 이해
  • → 구현 가능한 범위로 정리

"완벽히 아는 상태를 기다리지 않고, 움직이면서 이해합니다."

02

2️⃣ 핵심 강점 ③ 고객을 이해하는 개발자

강점 3. 고객을 이해하는 개발자

  • 실제 고객 응대 경험 보유
  • 사용자의 질문, 불만, 망설임 포인트를 알고 있음
  • 개발 시 '기술적 완성도'보다 '이해도'를 우선

사례

  • 결과 화면 문장 구조 단순화
  • 전문 용어 최소화
  • '이게 무슨 말인가요?'를 줄이는 방향으로 UX 설계

"개발자가 만든 것은 개발자가 이해할 수 있어야 하는 게 아니라, 사용자가 이해할 수 있어야 합니다."

03

3️⃣ 올해 주요 성과

📊 정량 성과

  • 사주 웹앱 핵심 기능 0 → MVP 구현
  • 주요 계산 로직 100% 코드화
  • 사용자 입력 → 결과 생성 엔드투엔드 플로우 완성

🤝 정성 기여

  • 기획/개념 논의를 개발 언어로 번역
  • 팀 내 기술 논의 시 구조적 정리 역할
  • 문제 발생 시 원인 분리 & 재현 중심 접근

4️⃣ 차별화 포인트

나만의 포인트

  • 기획에 관여하는 개발자
  • 고객 관점으로 개발 우선순위 조정
  • 기술 선택 시 비즈니스 영향 고려

"만들 수 있는 것보다, 팔릴 수 있는 것을 고민하는 개발자"

5️⃣ 협업 기여

  • 디자이너와 충돌 없이 구조적 대안 제시
  • 기획 논의 시 구현 가능성 기준 제시
  • 개발 일정과 품질의 균형을 맞추는 역할

팀의 시행착오 비용 감소

06

6️⃣ 2026년 성장 목표

핵심 목표

'매출을 만들어내는 개발자'

제품 관점

  • 기능 중심 → 문제 해결 중심
  • 사용자가 돈을 내는 지점에 집중
  • "좋은 기능"보다 "지불 이유" 설계

비즈니스 관점

  • 수익 구조 이해 (유료화 포인트, 전환)
  • 사용자 행동 → 매출로 이어지는 흐름 설계
  • 실험 → 데이터 → 개선 사이클 경험

개인 역량

  • 제품 기획 관점 강화
  • 고객 데이터 해석 능력
  • 영어 기반 글로벌 레퍼런스 지속 흡수
